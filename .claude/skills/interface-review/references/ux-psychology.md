# UI/UX設計におけるガイドライン

技術ブログにおけるUI/UXを設計・レビューする際は、以下の心理効果を考慮すること。

**基本方針**: コンテンツの質と明確性を最優先。マーケティング的な心理効果は控えめにする。

## 最優先で適用すべき効果

- **美的ユーザビリティ効果**: 美しいデザインは使いやすく感じられる。視覚的な洗練さを重視
- **認知負荷**: 情報量・レイアウト・タイポグラフィを最適化し、読者の負担を軽減
  - 技術記事では特にコードブロック、シンタックスハイライト、適切な空白に注意
- **視覚的階層**: 見出し・本文・コードの優先度を視覚的に明確化
  - 見出し構造、コードと本文の明確な区別を重視
- **ドハティの閾値**: ページ応答は400ms以内を目標。遅延は集中力低下を招く
  - 静的生成により初期表示の高速化を実現
- **バナー・ブラインドネス**: 広告的な見た目の要素は無視される。重要情報は広告風デザインを避ける
- **段階的開示**: 複雑な情報は段階的に表示。アコーディオンやタブを活用
- **系列位置効果**: ナビゲーション・リストの最初と最後に重要項目を配置
- **ピーク・エンドの法則**: 記事のハイライトと締めくくりに注力。全体印象を決定する

## 積極的に考慮すべき効果

- **フレーミング効果**: 同じ情報でも提示方法で印象が変わる。ポジティブな枠組みを選択
- **プライミング効果**: アイキャッチ・導入部が後続コンテンツの解釈に影響
- **選択的注意**: ユーザーの関心事項を視覚的に強調
- **ハロー効果**: 一貫したブランディングで全体評価を向上
- **誘導抵抗**: 過度なポップアップ・強制的な購読促進は反発を招く。控えめに
  - 技術ブログでは特に抑制的に
- **親近性バイアス**: 一般的なUIパターンを採用し学習コストを削減
  - 技術ブログにおいて特に重要
- **ビジュアル・アンカー**: スクロール時の視覚的固定点を設ける
- **決断疲れ**: 選択肢を適切に絞る。カテゴリ・メニュー項目は7±2を目安に
  - シンプルなナビゲーション構造を維持
- **デフォルト効果**: ダークモード・フォントサイズ等のデフォルト値を最適化
- **ユーザー歓喜効果**: マイクロインタラクションで予期せぬ喜びを提供

## 技術ブログで不要な効果

以下の効果は技術ブログの特性上、優先度が低いか不要:

- **好奇心ギャップ**: タイトルの明確性が最優先。クリックベイト的手法は信頼を損なう
- **社会的証明**: 技術的な正確性が信頼性の源泉。シェア数に依存する必要は低い
- **ツァイガルニク効果**: 各記事の完結性を重視。シリーズ記事が主要でない限り不要
- **ナッジ効果**: ニュースレター購読機能がなければ不要。過度なCTAは読者体験を損なう
- **労働の錯覚**: 静的生成により高速。ドハティの閾値を達成できればローディング演出は不要
